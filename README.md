# ☁️ CloudSieve — Complete Setup Guide

## Project Structure

```
cloudsieve/
├── backend/
│   ├── main.py              → FastAPI server (local mode)
│   ├── main_aws.py          → FastAPI server (local + AWS mode)
│   ├── engine.py            → Core pipeline: dedup, repair, CQI
│   ├── mlflow_tracker.py    → MLflow experiment tracking
│   ├── requirements.txt     → Python dependencies
│   ├── sample_data.csv      → Dirty test dataset
│   ├── .env.local           → Local environment config
│   └── .env.aws             → AWS environment config (generated by deploy.sh)
│
├── frontend/
│   ├── package.json         → React dependencies
│   ├── public/index.html    → HTML entry
│   └── src/
│       ├── index.js         → React entry point
│       ├── index.css        → Global styles
│       ├── App.js           → App shell with sidebar + routing
│       ├── App.css          → Layout styles
│       ├── api.js           → Axios API helper
│       └── pages/
│           ├── UploadPage.js/.css    → Drag & drop upload
│           ├── PipelinePage.js/.css  → Configure & run pipeline
│           ├── ResultsPage.js/.css   → Charts, CQI, tables
│           └── HistoryPage.js        → Past jobs
│
└── aws/
    ├── lambda_trigger.py    → S3 → Lambda → Glue trigger
    ├── glue_job.py          → PySpark ETL on AWS Glue
    ├── cloudformation.yaml  → Full AWS infrastructure as code
    └── deploy.sh            → One-command AWS deployment
```

## ✅ Prerequisites

Install these before starting:
- Python 3.9+   → https://python.org
- Node.js 18+   → https://nodejs.org
- AWS CLI       → https://aws.amazon.com/cli (only for Phase 2)

## 🚀 PHASE 1 — Local (No AWS Needed)

### Step 1 — Open VSCode
```
File → Open Folder → select the cloudsieve/ folder
```

### Step 2 — Open Terminal 1 (Backend)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```
You should see: `Uvicorn running on http://127.0.0.1:8000`

### Step 3 — Open Terminal 2 (Frontend)
```bash
cd frontend
npm install
npm start
```
Browser opens automatically at http://localhost:3000

### Step 4 — Test the App
1. Go to http://localhost:3000
2. Click Upload → drag in sample_data.csv from the backend/ folder
3. Click Pipeline → choose the 'name' column for fuzzy matching → Run
4. View Results → see CQI score, radar chart, fuzzy matches, repair log
5. Download your clean CSV

### Step 5 — Run MLflow Tracking
```bash
cd backend
pip install mlflow
python mlflow_tracker.py --file sample_data.csv --fuzzy_col name --threshold 85
mlflow ui
```
Open: http://127.0.0.1:5000 → see all logged metrics, params, and model

## ☁️ PHASE 2 — AWS Integration

### Prerequisites
```bash
# Install AWS CLI
pip install awscli

# Configure credentials
aws configure
# Enter: Access Key ID, Secret Access Key, Region (us-east-1), Output (json)
```

### Step 1 — Deploy AWS Infrastructure
```bash
cd aws
chmod +x deploy.sh
./deploy.sh
```
This automatically creates: S3 buckets, Lambda, Glue Job, DynamoDB, SNS, CloudWatch

### Step 2 — Upload Glue Script
```bash
# Already done by deploy.sh — but to do manually:
aws s3 cp aws/glue_job.py s3://cloudsieve-scripts-YOURACCOUNTID/scripts/glue_job.py
```

### Step 3 — Run Backend in AWS Mode
```bash
cd backend

# Load AWS environment variables
export $(cat .env.aws | xargs)

# Run with AWS integration
uvicorn main_aws:app --reload --port 8000
```

### Step 4 — Test AWS Pipeline
```bash
# Upload a file directly to S3 to test the Lambda trigger
aws s3 cp sample_data.csv s3://cloudsieve-raw-YOURACCOUNTID/

# Check Lambda was triggered
aws logs tail /aws/lambda/cloudsieve-trigger --follow

# Check Glue job started
aws glue get-job-runs --job-name cloudsieve-dedup-job
```

### Step 5 — Monitor in AWS Console
- Lambda logs:  CloudWatch → Log Groups → /aws/lambda/cloudsieve-trigger
- Glue runs:    Glue → Jobs → cloudsieve-dedup-job → Run history
- DynamoDB:     DynamoDB → Tables → cloudsieve-jobs → Explore items
- Dashboard:    CloudWatch → Dashboards → cloudsieve-dashboard

## 🔁 FULL INTEGRATION (Phase 3)

In this mode the frontend talks to the backend which uses AWS for storage.
The Glue job runs in parallel for large files on AWS.

### Run Backend (AWS Mode)
```bash
cd backend
export $(cat .env.aws | xargs)
uvicorn main_aws:app --reload --port 8000
```

### Run Frontend
```bash
cd frontend
npm start
```

The frontend is identical — it automatically uses AWS storage behind the scenes.
Upload → Pipeline → Results all work the same, but data flows through S3 + Glue + DynamoDB.

## ⚠️ Common Errors & Fixes

**Backend won't start — "module not found"**
```bash
pip install -r requirements.txt
```

**Frontend won't start — "npm not found"**
```bash
# Install Node.js from https://nodejs.org then retry
npm install && npm start
```

**CORS error in browser**
Make sure backend is running on port 8000 before starting frontend.

**AWS deploy fails — "credentials not configured"**
```bash
aws configure  # re-enter your credentials
```

**Glue job fails — "script not found"**
```bash
aws s3 cp aws/glue_job.py s3://cloudsieve-scripts-YOURACCOUNTID/scripts/glue_job.py
```
